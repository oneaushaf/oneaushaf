A little about Costan

Costan is a "kosan" management system, developed as the final project of "System Analisys and Design" subject, but considering real implementation of digital transition.

The product focuses on stating room conditions and managing transaction request in pseudo-realtime (a fully automated system would interupt some interaction in a digital transitioning environment) based on the performance of the admin. However, this app does not provide any payment gate so it works as the comunication intermediary.

Features and Capabilies

Stating room condition

Costan is able to provide all the room's state, such as Available, Occupied, or Under Maintenance. The behavior would also vary according to the room's state, like the room would not be able to be ordered if it is Occupied or Under Maintenance.

Managing Order, Room, and User

The order menu from this app would not only redirect the user to the admin chatroom, but it would also create an order request to the admin. The admin then can accept or decline the request based on the condition. Accpeting request would make the request owner a user with the requested username and some predefined password which can be changed later.

The web can also manage the registered user manually, so the order that does not come from the order menu (eg. contacting the admin directly instead of visiting the web), can still be organized by the admin. The admin and the user can also detect wether or not they have paid their bills based on the current date, so it would minimize postponded payment caused by the lack of detection and management

It also has some medium-level customization to the rooms, facility, and roomtype so the user can always be provided with up-to-date information

User account

Registered user will be given their own user account, used to track bill payments history, extending the rent duration, detecting unpaid bill, and contact to the admin for any inconvinient and request.
